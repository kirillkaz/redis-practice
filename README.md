## Запуск проекта
Для запуска проекта необходимо запустить docker-compose.yaml
То есть достаточно использовать

```sh
docker compose up
```

## Локальная разработка
Для локальной разработки достаточно прописать
```sh
uv sync
```

## Тестирование
Для тестирования проекта в корне проекта необходимо создать .env и скопировать
содержимое .env.example в .env

## Особенность реализации
После запуска проекта раз в короткий промежуток времени в redis будут помещаться фейковые заказы.

По умолчанию - все фейковые заказы пренадлежат временному промежутку 01.01.2026 - 31.01.2026
